# Tiers & Criteria

Two things give a ranking card authority: a **bounded, parallel tier system** and a **stated
basis**. Both go on the card. A ranking with no stated basis reads as a random opinion and
invites "你算老几"; a stated basis converts a hot take into judgment.

## Tier vocabularies

Pick ONE system per card. Keep labels parallel in form — all letters, or all nouns, never
mix "S" with "传奇". Use 3–6 tiers; 3–4 reads cleanest on a phone cover. More tiers means more
rows, smaller portraits, and a mushier card. Keep the top tier scarce (1–3 names) — that
scarcity is the payoff.

| System | Tiers (top → bottom) | Best for |
| --- | --- | --- |
| Letter (gamer-native) | S / A / B / C / D / (F) | Any domain; the default internet grammar. S sits above A. |
| Letter + plus | S+ / S / A / A- … | Fine gradation at the top (GOAT debates). Use sparingly. |
| 段位 (status) | 神 / 天王 / 一流 / 二流 / 三流 | 历史地位 content on CN platforms. "神" holds 1–3 max. |
| 巨星阶 (football) | 球王 / 准球王 / 超级巨星 / 巨星 / 球星 | Football legacy ranking; mirrors real fan taxonomy. |
| 水准 (quality) | 顶级 / 世界级 / 优秀 / 普通 / 平庸 | "顶级后卫 vs 普通后卫" position-quality cards. |
| GOAT pantheon | GOAT / 传奇 / 巨星 / 球星 / 角色 | Mixed-era greatness, basketball. |
| Numeric rank | 1, 2, 3 … N | Podium / grid / Top-10 where order matters more than buckets. |

Enforce: single vocabulary, parallel form, capped count, scarce top.

## Color ramp

Color-code tiers warm → cool, top → bottom, so rank reads before the labels do:

- S / 神 / GOAT — red `#ff5d5d`
- A / 天王 — orange `#ff9a3d`
- B / 一流 — yellow `#ffd23d` (dark text on it)
- C / 二流 — green `#5fd66b`
- D — blue `#43c5e0`
- F — purple / dark blue

Keep saturation consistent across bands. Warm = best, cool = worst is the grammar viewers
decode pre-reading.

This warm→cool ramp is the default for S/A/B/C and 段位 systems. For domains where viewers do
NOT read green as "worse than yellow" — cities, majors, quality bands (王牌/天坑, 一线/二线) —
a warm→cool ramp reads as a misleading pass/fail traffic light. There, use a **single-hue
value ramp** instead: one accent color, dark/saturated for the top tier fading to light for
the bottom, so lightness encodes rank without a false red-green signal.

## The ranking basis — credibility harness

Force the user to pick at least one primary criterion and PRINT it on the card. This is the
single biggest trust lever.

| 维度 | EN | Rewards | Bias |
| --- | --- | --- | --- |
| 历史地位 | Legacy | Longevity + cultural footprint | Favors older eras |
| 巅峰统治力 | Peak dominance | Best 2–5 year stretch | Favors short-peak flamethrowers |
| 荣誉 | Honors | Trophies, MVP, 金球, 总冠军 | Team-dependent |
| 数据 | Stats | Counting + advanced metrics | Era-inflated |
| 影响力 | Influence | Changed how the game/field is done | Subjective |
| 时代调整 | Era-adjusted | Normalizes across eras | The most defensible lever |
| 巅峰 vs 生涯 | Peak vs career | Forces a single axis | The classic 巅峰党 vs 生涯党 split |

## Printing the basis on the card

- One **basis line** under the title: `按巅峰统治力排名` or, with weights,
  `历史地位 = 荣誉40% · 巅峰35% · 影响力25%`. Weights/formula must reflect REAL rigor, not just
  look rigorous.
- **Basis-governs-tiers test (falsifiable).** Pick the 2 highest-ranked subjects and the 1
  most-debatable subject. For EACH, write the one-line reason its tier follows from the printed
  basis, citing the specific criterion value (荣誉 → trophy count, 巅峰 → peak-window dominance,
  数据 → the metric). If you cannot write that line for any of the three, or the line
  contradicts the basis (e.g. basis = 荣誉 but an S-tier name has fewer trophies than an A-tier
  name), the basis is not governing — move the subject, not the label. If the formula has
  weights, the top-tier ordering must match the weighted score; never print weights you did not
  compute. A faked formula is worse than none.
- Always print the **scope**: `仅限后卫` / `现役` / `历史` / `俱乐部生涯`. An unscoped ranking
  is the top cause of bad-faith arguments.
- If data-driven, add a tiny `数据来源：…` line.
- A small `个人向` / `一家之言` disclaimer preempts pile-ons without weakening the card.
