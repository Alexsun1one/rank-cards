# Prompt Records / 提示词留档

Use this reference for public examples, README/showcase cards, client work, or any ranking likely to be reused or challenged.

Prompt records make Rank Cards auditable — especially when exact order, scores, public-figure likeness, and domain register routing are involved.

## Minimal Record

Save or report:

```text
asset name:
source / ranking basis:
asset role: goat-pyramid / top-n / tier-list / before-after / carousel-page
exact order contract:
scores (if any):
scope:
style register:
layout archetype:
portrait pipeline:
truth constraints:
final prompt:
output path:
aspect ratio:
native text rendered:
rejected attempts:
QA result:
caption / source note:
```

## Rejected Attempts

Record a rejected attempt when it teaches a reusable lesson:

- invented or swapped ranks
- sports-only template lock on a non-sports domain
- unreadable / melted names
- wrong order vs exact-order contract
- AI plastic portraits or collage pipeline break
- missing basis/scope
- tier sprawl or mixed vocabulary
- wrong color encoding for the domain
- wrong-owner task that should have rerouted
- cover density failure at thumbnail size

Do not keep noisy notes for every ordinary draft.

## File Placement

For skill-package examples:

- put longer prompt records in `examples/` (e.g. `examples/prompts.md` or a task-specific record)
- put final images under the project's examples/images paths when the outer package has them

For article packages or external projects, use that project's asset folder.

## Why This Matters For Rank Cards

- Exact order is easy to drift under generation pressure — the record catches swaps.
- Domain register routing needs proof that non-sports cards were not forced into sports chrome.
- Public-figure cards need an audit trail for basis, scope, and likeness treatment.
- Native-text vs post-overlay decisions should be explicit for reuse.
